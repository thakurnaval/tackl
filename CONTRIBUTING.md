# Contributing

Thanks for contributing to this project.

## Getting Started

1. Fork the repository and create your branch from `main`.
2. Install dependencies:

   ```bash
   npm install
   ```

3. Start the app locally (see the README's "Local development" section for Firebase setup):

   ```bash
   npm run dev
   ```

## Development Guidelines

- Keep changes focused and small when possible.
- Follow existing code style and project structure.
- Prefer clear naming and simple, maintainable logic.
- Update documentation when behavior or workflows change.

## Reporting Issues

When opening an issue, include:

- A clear title and description.
- Steps to reproduce.
- Expected behavior and actual behavior.
- OS and app version/commit, if known.
- Screenshots or logs when helpful.

## Submitting Pull Requests

1. Ensure your branch is up to date with `main`.
2. Confirm the app runs locally after your changes.
3. Include a clear PR description:
   - What changed
   - Why it changed
   - Any trade-offs or follow-up work
4. Link related issues (for example, `Closes #123`).

## Commit Messages

Use clear, descriptive commit messages in the imperative mood.

Examples:

- `Fix window close behavior on macOS`
- `Add MIT license file`

## Review Expectations

- Be respectful and constructive.
- Address feedback with follow-up commits.
- If you disagree with feedback, explain your reasoning clearly.
